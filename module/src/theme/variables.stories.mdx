# Available SCSS variables

These are all the defaults for variables exported by Armstorng - you can override them yourself to update them and how they look throughough Armstrong.

They can be used after an import of:

```css
@import '~@rocketmakers/armstrong/dist/imports.scss';
```

(see setup/scss for more info)

## <a name="overriding">Overriding</a>

All of Armstrong's SCSS variables use `!default`, meaning they can be overriden from anywhere. Here's a recommended way of overriding them.

```css
/* in your own scss variables file, i.e. src/theme/variables.scss */
$brand-primary: red;
$spacing-large 500px;

/* in your theme entrypoint, i.e. src/theme/theme.scss */
@import '~@rocketmakers/armstrong/dist/imports.scss';
@import './variables.scss';

@import '~@rocketmakers/armstrong/dist/basic.scss';

.... the rest of your scss (see setup/scss)
```

## <a name="global">Global</a>

### Colors

```css
$black: #000
$white: #fff;
$brand-primary: #cd3939;
$brand-secondary: #3d4144;

$gray-base: #4f5c69;
$gray-very-dark: darken($gray-base, 20%);
$gray-dark: darken($gray-base, 15%);
$gray-medium: $gray-base;
$gray-light: lighten($gray-base, 50%);
$gray-very-light: lighten($gray-base, 60%);

$color-positive: #00a388;
$color-warning: #f78e52;
$color-negative: #da2e18;
$color-info: #3498d8;
```

### Spacing

```css
$spacing-xxsmall: 0.25rem;
$spacing-xsmall: 0.5rem;
$spacing-small: 1rem;
$spacing-medium: 2rem;
$spacing-large: 3rem;
$spacing-xlarge: 4rem;
```

### Border Radiuses

```css
$border-radius-small: $spacing-xxsmall;
$border-radius-medium: $spacing-small;
$border-radius-large: $spacing-medium;
```

### Shadows

```css
$shadow-opacity: 0.25;
$shadow-color: rgba($gray-very-dark, $shadow-opacity);

$shadow-size-xsmall: 0.2rem;
$shadow-size-small: 0.5rem;
$shadow-size-medium: 1rem;
$shadow-size-large: 2rem;
$shadow-size-xlarge: 3rem;

$shadow-offset-xsmall: 0.05rem 0.05rem;
$shadow-offset-small: 0.1rem 0.1rem;
$shadow-offset-medium: 0.2rem 0.2rem;
$shadow-offset-large: 0.4rem 0.4rem;
$shadow-offset-xlarge: 1.5rem 1.5rem;

$shadow-xsmall: $shadow-offset-xsmall $shadow-size-xsmall $shadow-color;
$shadow-small: $shadow-offset-small $shadow-size-small $shadow-color;
$shadow-medium: $shadow-offset-medium $shadow-size-medium $shadow-color;
$shadow-large: $shadow-offset-large $shadow-size-large $shadow-color;
$shadow-xlarge: $shadow-offset-xlarge $shadow-size-xlarge $shadow-color;

$shadow-inner-xsmall: $shadow-offset-xsmall $shadow-size-xsmall $shadow-color inset;
$shadow-inner-small: $shadow-offset-small $shadow-size-small $shadow-color inset;
$shadow-inner-medium: $shadow-offset-medium $shadow-size-medium $shadow-color inset;
$shadow-inner-large: $shadow-offset-large $shadow-size-large $shadow-color inset;
$shadow-inner-xlarge: $shadow-offset-xlarge $shadow-size-xlarge $shadow-color inset;
```

## <a name="components">Components</a>

More specific variables used in components which can be overriden to make restyling them a bit easier, usually for values that are repeated throughought the component.

Not all components need to use scss variables other than the global ones and should be more self explanatory to override through inspection.

### CalendarDisplay

```css
/* the width of an armstrong calendar */
$arm-calendar-display-width: 300px;

/* the size of the coloured circle behind active dates */
$arm-calendar-display-date-circle: 1.6rem;

/* the size of the highlight dot below a date */
$arm-calendar-display-date-highlight-size: 0.4rem;
```

### Dialog

```css
/* the width of an armstrong dialog */
$dialog-width: 500px;

/* the maximum height of an armstrong dialog */
$dialog-height: calc(100vh - #{$spacing-medium * 2});
```

### Dropdown

```css
/* minimum width of the armstrong dropdown */
$arm-dropdown-min-width: 50px;

/* maximum height of the armstrong dropdown */
$arm-dropdown-height: 240px;
```

### ProgressBar

```css
/* the thickness of the progress bar */
$arm-progress-bar-thickness: 0.5rem;

/* the thickness of the progress bar if it's been given a label prop */
$arm-progress-bar-with-label-thickness: 1rem;
```

### RangeInput

```css
/* the height of the range input's track */
$arm-range-input-track-thickness: 6px;
```

### SideMenu

```css
/* the width of the side menu */
$arm-side-menu-width: 240px;
```

### Stepper

```css
/* the size of the icon in a stepper */
$arm-stepper-icon-height: 2.2rem;

/* the size of the icon if no icon or index is provided in a stepper */
$arm-stepper-icon-height-small: 1rem;
```

### SwitchInput

```css
/* the width of the switch */
$arm-switch-input-width: 50px;

/* the height of the switch */
$arm-switch-input-height: 30px;

/* the padding for the handle in the switch input */
$arm-switch-input-handle-padding: math.div($spacing-xxsmall, 2);

/* the size of the handle, by default calculated using the given height and padding */
$arm-switch-input-handle: calc(#{$arm-switch-input-height} - #{$arm-switch-input-handle-padding * 2});
```

### TimeInput

```css
/* the width of the time input */
$arm-time-input-width: 50px;
```

### ToastNotification

```css
/* the width of the toast noficiation */
$arm-toast-notification-width: 200px;
```

### Tooltip

```css
/* the max width of the tooltip */
$arm-tooltip-max-width: 250px;
```

### Header

```css
/* the height of the header */
$arm-header-height: 60px;
```

## <a name="animations">Animations</a>

These are variables which dictate the animations described in `SCSS/animations`

```css
$slide-amount-large: 30px;
$slide-amount-medium: 20px;
$slide-amount-small: 10px;

$scale-amount-large: 0.3;
$scale-amount-medium: 0.2;
$scale-amount-small: 0.05;

$rotate-amount-large: 360deg;
$rotate-amount-medium: 180deg;
$rotate-amount-small: 90deg;
$rotate-amount-very-small: 45deg;

$expand-max-small: 100px;
$expand-max-medium: 300px;
$expand-max-large: 700px;
```
